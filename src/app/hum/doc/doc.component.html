Ma définition :

La rédaction de documentation est une compétence importante car elle permet de guider au mieux les utilisateurs sur l’utilisation d’une application dans le cas d’une documentation utilisateur mais également les développeurs pour comprendre comment la reprendre afin de pouvoir l’améliorer dans le cas d’une documentation technique et comprendre les intentions initiales.


Mes éléments de preuve :

Lors de mon stage dans le cadre du projet Open ENT (lien vers la réalisation), j’ai travaillé sur la documentation technique de mon module afin de la mettre à jour et qu’elle soit en accord avec ce qui est présent sur l’application. J’ai travaillé sur la documentation API durant ce dernier et découvert Swagger qui est une plateforme (CHANGER LA DEFINITION DE SWAGGER) permettant de réaliser la documentation d’une API en utilisant le YAML. Dans cette dernière, j’ai dû référencer les différents points d'entrée de l’API (endpoints) avec leurs paramètres d'entrée mais également les réponses envoyées en fonction de si cela se passait bien ou pas. Lors de mes PR sur Github, je devais renseigner si la documentation avait été mise à jour et dans le cas contraire expliquer pourquoi elle ne l’avait pas été. Dans le projet Grand Angle, j’ai dû Lors de projets réalisés dans le cadre scolaire comme LinkNTrade, j’ai participé à la réalisation des documentations utilisateur et technique mais j’ai également réalisé dans le projet Geosavior, une documentation pour expliquer le processus d’installation d’une application android avec un fichier apk sur un émulateur. En effet, la commanditaire ne possédait pas de smartphone et ne pouvait donc pas installer l’application pour pouvoir la tester. J’ai alors rédigé un document pour expliquer pas à pas comment installer l’émulateur BlueStack et comment installer l’apk contenant l’application.
