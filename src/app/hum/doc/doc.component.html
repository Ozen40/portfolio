Rédaction de documentation

Ma définition :

La rédaction de documentation est une compétence importante car elle permet de guider au mieux les utilisateurs sur l’utilisation d’une application dans le cas d’une documentation utilisateur mais également les développeurs pour comprendre comment la reprendre afin de pouvoir l’améliorer dans le cas d’une documentation technique et comprendre les intentions initiales.


Mes éléments de preuve :

Lors de mon stage dans le cadre du projet OpenENT, j’ai travaillé sur la documentation technique de mon module afin de la mettre à jour et qu’elle soit en accord avec ce qui est présent sur l’application.

J’ai travaillé sur la documentation API durant ce dernier et découvert Swagger qui est une plateforme permettant de réaliser la documentation d’une API en utilisant le YAML. Dans cette dernière, j’ai dû référencer les différents points d'entrée de l’API (endpoints) avec leurs paramètres d'entrée mais également les réponses envoyées en fonction de si cela se passait bien ou pas.

Lors de mes PR sur Github, je devais renseigner si la documentation avait été mise à jour et dans le cas contraire expliquer pourquoi elle ne l’avait pas été. Dans le projet Grand Angle, j’ai dû

Lors de projets réalisés dans le cadre scolaire comme Link & Trade, j’ai participé à la réalisation des documentations utilisateur et technique mais j’ai également réalisé dans le projet Geosavior, une documentation pour expliquer le processus d’installation d’une application android avec un fichier apk sur un émulateur. En effet, la commanditaire ne possédait pas de smartphone et ne pouvait donc pas installer l’application pour pouvoir la tester. J’ai alors rédigé un document pour expliquer pas à pas comment installer l’émulateur BlueStack et comment installer l’apk contenant l’application.


Mon autocritique :

Je me considère comme ayant un niveau autonome dans la rédaction de documentation grâce à mon travail sur plusieurs projets scolaires qui demandaient la réalisation de documentations utilisateur et technique mais également grâce aux différentes évolutions réalisées durant mon alternance car une documentation devait être rédigée pour chacune d’elles.
J’ai cependant du mal avec la mise en forme de ma documentation lorsqu’elle devient conséquente.
Apprendre à rédiger une documentation est important car il permet d’exprimer de manière claire et de lister des fonctionnalités d’une application.


Mon évolution dans cette compétence :

J’ai pour objectif d’atteindre le niveau confirmé en rédaction de documentations. Je vais donc apprendre le markdown car il est très utilisé pour faire de la mise en forme de documentations comme dans Gitlab par exemple.
Pour m’améliorer dans la rédaction de documentation d’API, j’ai prévu de suivre le guide “ Documenting APIs: A guide for technical writers and engineers ” pour mieux apprendre à documenter une API.
